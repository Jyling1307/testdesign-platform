            - UI
              
                - 只读、读写、读写删和完全控制选项：  
                  1、存储服务：cifs（用户和用户组）、ftp（用户和用户组）、nfs权限处  
                  2、存储管理-桶详情-共享管理中
                  
                    - 权限排序 
                      
                        - 四种权限从左到右、从大到小排序，依次为：“完全控制”、“读写删”、“读写”、“只读”。
                          
                    - 权限说明（格式待定）
                      
                        - 完全控制  
                          1、包括读写删，chmod/chown/修改acl
                          
                        - 读写删  
                          1、支持删除，重命名  
                          2、不支持chmod，chown，acl等涉及权限控制的属性及扩展属性）
                          
                        - 读写  
                            
                          1、支持读写数据，覆盖写，truncate等操作  
                          2、支持修改扩展属性和属性  
                          3、不支持chmod，chown，acl等涉及权限控制的属性及扩展属性  
                          4、不支持删除，重命名
                          
                        - 只读  
                          1、支持数据和元数据只读  
                          2、不支持任何数据修改或元数据修改权限
                          
                - 页面中权限过滤筛选、修改权限后与配置一致

                                - 预期以各自访问协议中共享权限配置为准，没有丢失或io中断
                                  
                        - 小到大
                          
                            - nfs客户端权限从读写删-->完全控制，cifs客户端从读写-->读写删，ftp客户端权限从只读-->读写
                              
                                - 预期以各自访问协议中共享权限配置为准，没有丢失或io中断

                    - 比如写到一半，将权限从读写切换到只读，保留当前open文件的操作，重做open时不可写

            - 持续io，分别故障三类文件网关服务（nfs-ganesha、ftp、samba），读写失败
